Pac Labyrinth

This project has been put on the back burner while I shifted my goals towards learning web development. I do plan on returning to it in the future.

This is a huge undertaking that involves a variety of disciplines. I believe it showcases my versatility and broad range of interests.
It involves 3D printing, microcontroller (arduino) coding, PCB manufacturing, video game developement with the game engine Godot using GDScript and more...

Inspiration

I had an old game growing up that had a ball in a maze that you had to tilt using dials to control. Combining this with pac man style dot eating gives the inspiration for my game controller.

Construction

  • PCBs

I made my own PCBs for this project by etching with ferric chloride. The ball (steel bearing) connects between pads on opposite sides as it rolls down the channels.

  • 3D Printed Prototype

I modeled the board in Blender, printed out the pieces for the prototype controller and glued the PCBs in place. I laser cut the box that the 3D printed portion sits on top of at HackRVA. Two buttons, a switch and a power jack are included in the prototype.

  • Arduino

I wrote code in C for an Arduino microcontroller for the games. The pads on the PCBs are all multiplexed so a minimum number of pins can be used to control them.

  • Godot

I made a game in Godot to integrate the controller in an augmented reality experience. There is also a 2 player game that can be seen at the end of the video.

  • Accelerometer

The controller will have an accelerometer in place to assist with syncing the controller and the Godot game. As a stand in, I used the accelerometer found in modern phones and have a separate controller application also written in Godot.
